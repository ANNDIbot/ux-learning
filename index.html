                        <h3 class="text-xl font-semibold mb-4">用户追求满意 (Satisficing)，而非最佳</h3>

                        <p class="text-gray-600 mb-6">用户不会花大量时间去比较所有选项，以找到“最优解”。一旦找到一个“足够好”的、能解决当前问题的方案，他们就会停下来并点击。这种行为被称为“满意即可”（Satisficing）。</p>

                        <h3 class="text-xl font-semibold mb-4">用户乐于试错 (Trial and Error)，而非理解</h3>

                        <p class="text-gray-600 mb-6">当不确定如何操作时，用户更倾向于直接点击尝试，而不是去阅读冗长的说明文档。好的设计应允许用户安全地试错、快速获得反馈并轻松返回。</p>

                    <p class="section-subtitle text-gray-500">遵循这些原则，可以帮助我们创造出更直观、易用的产品。点击下方卡片，查看每个原则的详细解释和示意图。</p>

                    <h2 class="section-title font-bold mb-4">可用性测试 (Usability Testing)</h2>

                    <p class="section-subtitle text-gray-500">如果你想让产品变得更好，测试是无可替代的。好消息是，测试可以非常简单、廉价且有效。</p>

                    <p class="section-intro-text text-gray-700 max-w-3xl mx-auto mt-6 mb-12">可用性测试是发现产品中“别让我思考”问题的最有效方法。它能让你从真实用户的角度，发现那些你自己可能永远无法察觉的问题，这也被称为“知识的诅咒 (Curse of Knowledge)”。</p>

                    <h3 class="text-xl font-semibold text-center mb-6">简单的可用性测试流程 (Simple Usability Testing Process - 5 Steps)</h3>

                    <p class="text-center text-gray-600 mb-8">可用性测试无需复杂，遵循以下简单步骤即可：</p>

                    <div class="flex flex-col items-center space-y-6">

                        <div class="flex flex-col md:flex-row items-center space-y-4 md:space-y-0 md:space-x-8">

                            <div class="diagram-box flex-shrink-0">1. 准备 (Prepare)</div>

                            <div class="arrow"></div>

                            <div class="diagram-box flex-shrink-0">2. 招募 (Recruit)</div>

                            <div class="arrow"></div>

                            <div class="diagram-box flex-shrink-0">3. 执行 (Execute)</div>

                        </div>

                        <div class="flex flex-col md:flex-row items-center space-y-4 md:space-y-0 md:space-x-8">

                            <div class="diagram-box flex-shrink-0">5. 改进 (Improve)</div>

                            <div class="arrow"></div>

                            <div class="diagram-box flex-shrink-0">4. 分析 (Analyze)</div>

                        </div>

                        <p class="text-sm text-gray-500 mt-6">（箭头表示流程方向）</p>

                    </div>

                    <ul class="list-disc list-inside text-gray-700 mt-10 space-y-3">

                        <li>**准备 (Prepare)：** 明确测试目标，设计任务场景。</li>

                        <li>**招募 (Recruit)：** 找到3-5名目标用户（不必是专家）。</li>

                        <li>**执行 (Execute)：** 让用户完成任务，观察他们的行为，倾听他们的思考，不要引导或干预。</li>

                        <li>**分析 (Analyze)：** 记录并总结发现的问题，特别是用户感到困惑或出错的地方。</li>

                        <li>**改进 (Improve)：** 根据发现的问题，迭代优化你的设计。</li>

                        <li>**关于“忙碌用户测试” (The Fat Man Test)：** 这是书中一个比喻，旨在强调设计应让最“不愿思考”的用户也能轻松使用。其核心是确保设计普适且直观。</li>

                    </ul>

                    <h2 class="section-title font-bold mb-4">移动端设计 (Mobile Usability)</h2>

                    <p class="section-subtitle text-gray-500">“别让我思考”的原则在移动端同样适用，甚至更为重要，因为移动设备的使用场景和物理限制带来了新的挑战。</p>

                    <p class="section-intro-text text-gray-700 max-w-3xl mx-auto mt-6 mb-12">移动设备的小屏幕、触摸操作和碎片化使用场景，要求设计师更加注重内容的精简和交互的直观性。</p>

                        <div class="w-full h-1/6 bg-gray-200 flex items-center justify-center text-xs text-gray-600 rounded-md mb-3">次要信息B (Secondary Info B)</div>

                    <ul class="space-y-4 text-gray-700 list-disc list-inside max-w-md">

                        <li><strong class="font-semibold">屏幕更小 (Smaller Screens)：</strong>必须做出取舍，只保留最重要的内容和功能。避免信息过载。</li>

                        <li><strong class="font-semibold">触摸操作 (Touch Interaction)：</strong>按钮和链接需要足够大（建议至少44x44像素），方便手指点击，避免误触。</li>

                        <li><strong class="font-semibold">碎片化使用 (Fragmented Use)：</strong>用户可能在任何地方、任何时候使用你的产品。设计需要支持快速、短时的交互，让用户能迅速完成任务。</li>

                        <li><strong class="font-semibold">上下文更重要 (Context is Key)：</strong>移动设备可以利用用户的位置、时间和方向等上下文信息，提供更智能、个性化的服务。</li>

                        <li><strong class="font-semibold">减少输入 (Minimize Input)：</strong>尽量减少用户在移动设备上的文字输入，提供选项、自动填充等便捷方式。</li>

                let currentActiveSectionId = 'home'; 


                for (let i = sections.length - 1; i >= 0; i--) {

                    const section = sections[i];

                    // Adjusted offset for sticky header and visual comfort

                navLinks.forEach(link => {

                });

                if (activeLink) {

                    activeLink.classList.add('active');

                }

                

                mobileNav.value = `#${currentActiveSectionId}`;

            }

            updateActiveLink();

                    mobileNav.value = targetId;

                });
